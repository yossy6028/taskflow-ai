# TaskFlow AI - 改善履歴

## 2024年 プロジェクト開始

### 初期設計の決定事項
- **データ保存**: ローカル（SQLite）とクラウド（Firebase）のハイブリッド構成
- **認証**: 個人利用のため認証機能は実装しない（将来の拡張性は考慮）
- **同期**: ローカル優先で、クラウドは自動バックアップとして活用

### 技術選定の経緯
- 当初検討した技術スタックと最終決定の差異を記録予定
